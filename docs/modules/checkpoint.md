# Check Point Firewall

**Dataset:** `check_point_vpn_1_firewall_1_raw` (default), `check_point_smartdefense_raw` (SmartDefense events), `check_point_app_control_raw` (App Control events)
**Transport:** Syslog (TCP)
**Format:** CEF (ArcSight Common Event Format)

Simulates Check Point R81.10 gateway logs covering stateful firewall traffic, IPS/Threat Prevention, Identity Awareness, URL Filtering, Application Control, SmartDefense, and VPN events. Dataset routing is controlled by the `device_product` field in the CEF header: the default `VPN-1 & FireWall-1` routes to `check_point_vpn_1_firewall_1_raw`; `SmartDefense` routes to `check_point_smartdefense_raw`; `Application Control` routes to `check_point_app_control_raw`.

> **Pending:** URL Filtering events (`cefDeviceEventClassId=url_filtering`) and Identity Awareness events (`identity`) technically belong to `check_point_url_filtering_raw` and `check_point_identity_awareness_raw` respectively. Without a `device_product` override these still land in `check_point_vpn_1_firewall_1_raw`.

## Benign Events

Benign traffic is generated by `_generate_benign_log`, which selects from seven log types using weighted random choice. All events are formatted by `_format_checkpoint_cef` which automatically appends `loguid`, `origin`, `rt`, `session_id`, `layer_name`, `layer_uuid`, and `blade` fields. The `cs1`/`cs1Label` fields carry the policy rule name on all traffic events. `suser` and `shost` carry the resolved username and hostname from session context.

## Threat Events

Threat events are dispatched from `_generate_threat_log`, which resolves a user/IP from session context and selects a threat type by weighted random choice from `_DEFAULT_THREAT_EVENTS`. Multi-event generators (noted below) return a Python list of CEF strings. SmartDefense and App Control events pass a `device_product` override to the CEF formatter for correct XSIAM dataset routing.
